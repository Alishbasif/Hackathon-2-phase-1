## ğŸ§  Development Approach (Spec-Driven)
Each feature in Phase I follows this workflow:

1. **Markdown Constitution**  
   Defines global rules, constraints, and architectural boundaries.

2. **Feature Specification (Spec)**  
   Describes behavior, inputs, outputs, and edge cases.

3. **Gemini CLI Code Generation**  
   Gemini CLI generates the implementation strictly from the spec.

4. **Spec Refinement**  
   Specs are iteratively improved until the generated output meets requirements.
